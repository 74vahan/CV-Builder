# 03. Шаблоны (стили дизайна CV)

## Как устроено
Предпросмотр всегда генерирует ОДНУ И ТУ ЖЕ семантическую разметку (preview.js):
`.cv > .cv-header (фото, имя, должность, .cv-contacts) + .cv-body (.cv-section …)`.
Стиль задаётся классом `tpl-<name>` на корневом `#cv`. Общий каркас листа
(размер A4, сброс отступов, `.cv-photo img`, `.cv-entry`) лежит в `css/base.css`
в блоке «Shared CV sheet frame».

## Размер листа
`.cv` — фиксированный A4 в пикселях: `width: 794px; min-height: 1123px`
(210×297 мм при 96 dpi). Фиксированный размер важен для точного соотношения
сторон при экспорте в PDF (см. 04-export). НЕ возвращать `mm` + `max-width:100%`
— из-за сжатия по ширине ломается пропорция и появляется лишняя страница.

## Четыре стиля (по файлу на каждый, `css/templates/`)
- `classic.css` — «Классический»: центрированная шапка, круглое фото, одна колонка.
- `modern.css` — «Современный»: `.cv-header` становится тёмным сайдбаром слева
  (flex, 34%), контент справа. Акцент #2f6bff.
- `minimal.css` — «Минимализм»: много воздуха, тонкий шрифт, без цвета, шапка в ряд.
- `elegant.css` — «Элегантный»: серифные заголовки (Georgia), тёплый акцент
  #c9a86a, центрированные секции.

## Акцентный цвет (var(--cv-accent))
Каждый шаблон в местах своего акцента использует `var(--cv-accent, <родной цвет>)`.
Пока пользователь не выбрал цвет — работает fallback (родной вид сохраняется). При
выборе цвета preview.js ставит `--cv-accent` на `#cv`, и акценты всех шаблонов
перекрашиваются. Новые accent-точки в шаблоне добавлять только через эту переменную.

## Ограничение html2canvas
CSS шаблонов держать в пределах того, что корректно снимает html2canvas 1.4.1:
обычные hex-цвета (НЕ `oklch`/`lab`), flexbox, простые тени/бордеры. Никаких
внешних шрифтов и сложных `filter` — иначе PDF разъедется.

## Как добавить новый стиль
1. Создать `css/templates/<name>.css`, все правила скоупить под `.cv.tpl-<name>`.
2. Подключить файл в `index.html`.
3. Добавить кнопку `<button class="chip" data-template="<name>" data-i18n="style.<name>">`
   в топбар и ключ `style.<name>` во все словари i18n.
4. Добавить `<name>` в массив `TEMPLATES` в `js/templates.js`.
