# 02. UI и локализация

## Раскладка
- Топбар (`.topbar`, класс `no-print`): название, выбор стиля (chips), пикер
  акцентного цвета, выбор языка, кнопки «Скачать HTML», «Скачать PDF», «Очистить».
  Полупрозрачный фон + `backdrop-filter`.
- `.layout` — грид на 2 колонки: слева редактор (`.editor`, скроллится), справа
  предпросмотр (`.preview-wrap` центрирует лист `.cv`).
- На ширине < 900px колонки складываются в одну.

## Дизайн, анимации, доступность
- Дизайн-токены (цвета, тени, радиусы, `--ease`) — в `:root` в `css/base.css`.
- Анимации вынесены в `css/animations.css`: появление блоков/листа, pop у новых
  элементов списков, «swap» при смене стиля. Всё под
  `@media (prefers-reduced-motion: no-preference)` + жёсткий стоп в `reduce`.
- ВАЖНО: анимация смены стиля висит на `#cv` временно (класс `.cv-switch`) и
  снимается по `animationend` + перед экспортом — иначе html2canvas снял бы
  прозрачный кадр (пустой PDF). См. 04-export.
- Доступность: `:focus-visible` обводка, aria-подписи на пикере акцента/языке.

## Акцентный цвет
- `state.accent` ('' = дефолт шаблона). Пикер `#accent-input` + сброс
  `#accent-reset` (main.js `initAccent`). preview.js ставит `--cv-accent` на `#cv`
  (или снимает при ''). Шаблоны используют `var(--cv-accent, <родной цвет>)`.

## Локализация (i18n)
- Три языка: en, ru, hy. Дефолт — ru.
- Словари: `i18n/en.js`, `i18n/ru.js`, `i18n/hy.js` → `window.CVApp.i18n.dict.<lang>`.
  Ключи плоские, вида `section.personal`, `field.fullName`, `cv.experience`.
- В разметке НЕ хардкодим тексты. Используем атрибуты:
  - `data-i18n="key"` — текст элемента,
  - `data-i18n-ph="key"` — placeholder,
  - `data-i18n-title="key"` — title.
- `i18n.apply(root)` проставляет тексты по этим атрибутам (работает и на
  динамически созданных элементах списков).
- Заголовки секций ВНУТРИ CV (Профиль/Опыт/…) тоже переводятся: preview.js берёт
  их через `i18n.t('cv.*')`, поэтому смена языка перерисовывает и предпросмотр
  (селектор языка пишет `state.lang` → notify → render).

## Исключение по кириллице
В `index.html` в `<select id="lang-select">` названия языков — эндонимы
(«Русский», «Հայերեն», «English»). Это осознанно и не нарушает правило про
случайную кириллицу: языковой переключатель всегда показывает названия на самих
языках.

## Как добавить новый ключ перевода
Добавить ключ во ВСЕ три словаря (en/ru/hy) и повесить `data-i18n` на элемент
(или использовать `t('key')` в JS). en служит фолбэком, если ключа нет в языке.
