# 04. Экспорт

Файл: `js/export.js` (`CVApp.exporter`). Библиотеки лежат локально в
`js/vendor/` (html2canvas 1.4.1, jsPDF 2.5.2 UMD) — интернет при работе не нужен.

## PDF в один клик (основной способ)
Кнопка `#btn-pdf` → `toPdf()`:
1. `html2canvas(#cv, {scale: 2, backgroundColor: '#fff'})` — снимок листа в canvas.
2. `new jspdf.jsPDF('p','mm','a4')`, картинка вписывается по ширине (210 мм),
   высота считается пропорционально.
3. Если контент выше страницы (297 мм) — режется на несколько страниц A4
   сдвигом изображения (`position -= pageH`).
4. `pdf.save('<Имя>.pdf')`.

Имя файла — из `fullName` (санитизировано), иначе `t('export.emptyName')` = "CV".

### Важный нюанс: анимации и html2canvas
html2canvas клонирует `#cv` и рендерит его заново. Если на `#cv` в этот момент
висит CSS-анимация (класс `.cv-switch` при смене стиля) — клон снимет её НАЧАЛЬНЫЙ
кадр (opacity:0) → пустой/смещённый PDF. Поэтому класс снимается по `animationend`
(preview.js) и дополнительно перед захватом в `toPdf()` (export.js). Не вешать
постоянные анимации на `#cv` и его содержимое.

### Важный нюанс: допуск EPS
В цикле разбивки страниц стоит `EPS = 1` мм: `while (heightLeft > EPS)`. Без него
субмиллиметровое округление (imgH ≈ 297.02 мм при точном A4) добавляло пустую
вторую страницу. НЕ убирать этот допуск.

## HTML-экспорт (запасной)
Кнопка `#btn-html` → `toHtml()`: собирает CSS из `document.styleSheets`
(`collectCss()`, best-effort, cross-origin листы пропускаются), оборачивает
`#cv.outerHTML` в самодостаточный HTML и отдаёт через Blob-ссылку. По `file://`
сбор CSS может частично не сработать — основной путь всё равно PDF.

## Проверено
- Валидный `data:application/pdf`, соотношение сторон ровно A4 (1.414).
- Короткое CV → 1 страница; длинное → корректно несколько.
- Фото (base64) попадает и в предпросмотр, и в PDF.
