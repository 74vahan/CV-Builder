# CV Builder — конструктор резюме

Статичный сайт, где пользователь заполняет данные о себе, выбирает один из
нескольких стилей дизайна, добавляет фото и **скачивает готовое резюме в PDF**.
Всё работает прямо в браузере — без сервера и без отправки данных куда-либо.

## Возможности
- Форма: личные данные, «о себе», опыт работы, образование, навыки, языки
  (блоки опыта/образования/языков можно добавлять и удалять).
- Загрузка фото (хранится локально, попадает в PDF).
- 4 стиля дизайна: Классический, Современный, Минимализм, Элегантный.
- **Выбор акцентного цвета** резюме (пикер в топбаре) — перекрашивает акценты
  выбранного стиля; сброс возвращает родной цвет шаблона.
- Интерфейс на 3 языках: English, Русский, Հայերեն (меняются и подписи, и
  заголовки секций внутри резюме).
- Экспорт: **PDF в один клик** (основное) и HTML (запасной вариант).
- Черновик автоматически сохраняется в браузере (localStorage) и восстанавливается
  при следующем открытии. Кнопка «Очистить» сбрасывает форму.
- Плавные анимации и микро-взаимодействия, полностью отключаются при системной
  настройке «уменьшить движение» (`prefers-reduced-motion`).
- Доступность: видимый фокус с клавиатуры (`:focus-visible`), aria-подписи.

## Как запустить локально
Проект — обычная статика, сборка не нужна.

**Вариант 1 (проще всего):** открыть файл `index.html` двойным кликом в браузере.

**Вариант 2 (локальный сервер, рекомендуется — корректнее работает HTML-экспорт):**
```bash
# из папки проекта
python -m http.server 5050
# затем открыть http://localhost:5050
```
или через Node:
```bash
npx --yes serve -l 5050 .
```

## Как задеплоить
Это статика — подойдёт любой статический хостинг. Достаточно выложить содержимое
папки (`index.html`, `css/`, `js/`, `i18n/`) как есть:
- GitHub Pages / GitLab Pages,
- Netlify / Vercel (drag&drop папки),
- любой nginx/Apache, отдающий статические файлы.
Сборка и переменные окружения не требуются.

### Свой сервер в GCP + CI/CD
В репозитории есть готовая инфраструктура: **Terraform** (`infra/`) поднимает VM в
Google Cloud с nginx, а **GitHub Actions** (`.github/workflows/deploy.yml`) при
пуше в `master` автоматически заливает статику по SCP и перезагружает nginx.
Пошаговый runbook — в [DEPLOY.md](DEPLOY.md).

## Структура
```
index.html            — разметка страницы и подключение скриптов
css/
  base.css            — токены дизайна, каркас, топбар, кнопки, общий лист CV (A4)
  editor.css          — стили формы-редактора
  animations.css      — анимации + guard для prefers-reduced-motion
  print.css           — правила печати
  templates/*.css     — 4 стиля дизайна CV (акцент через var(--cv-accent))
js/
  state.js            — данные + localStorage
  i18n.js             — движок локализации
  photo.js            — загрузка фото
  fields.js           — динамические списки (опыт/образование/языки)
  form.js             — связывание полей формы со state
  preview.js          — генерация предпросмотра CV
  templates.js        — переключение стилей
  export.js           — экспорт PDF/HTML
  main.js             — инициализация
  vendor/             — html2canvas, jsPDF (локально)
i18n/                 — словари en / ru / hy
infra/                — Terraform: VM в GCP + nginx (см. DEPLOY.md)
.github/workflows/    — CI/CD: автодеплой на сервер
memory/               — заметки по проекту (см. memory/INDEX.md)
```

## Технологии
Vanilla JS (без фреймворков и сборки), [html2canvas](https://html2canvas.hertzen.com/)
и [jsPDF](https://github.com/parallax/jsPDF) для экспорта в PDF (лежат локально в
`js/vendor/`, интернет при работе не требуется).

## Приватность
Все данные (включая фото) обрабатываются и хранятся только в вашем браузере.
Ничего не отправляется на сервер.
